# Claude Usage Widget

Небольшой десктопный виджет для Linux, который держится поверх окон и показывает,
сколько ресурсов Claude Code вы израсходовали:

- **Кольца лимитов** — % использования текущей сессии (5-часовое окно) и недельных
  лимитов «Все модели» и «Sonnet».
- **Токены** — входные/выходные/всего за сегодня или за неделю, с разбивкой по
  моделям и числом запросов.
- **История** — графики потребления токенов и пиковых лимитов по дням (7 / 14 дней).
- Светлая и тёмная тема, кнопка ручного обновления, тултипы.

## Требования

- **Linux** на базе apt (Ubuntu / Debian / Mint и т.п.). На других дистрибутивах
  установщик отработает, но системные пакеты придётся поставить вручную.
- **Python 3** с модулем `venv`.
- **Claude Code CLI** — установлен, доступен в `PATH` (или в `~/.local/bin/claude`)
  и уже залогинен (виджет рассчитан на план Max). Без рабочего `claude` кольца
  лимитов показывать нечего.
- Графика: GTK3 + WebKit2GTK (ставит установщик).

## Установка

Из папки с проектом:

```bash
./install.sh
```

Скрипт сделает следующее:

1. Поставит системные пакеты: `python3-venv`, `python3-gi`, GTK3 и WebKit2GTK
   (`gir1.2-webkit2-4.1` или `-4.0`).
2. Скопирует приложение в `~/.local/share/claude-widget`.
3. Создаст виртуальное окружение там же и установит `pywebview`.
4. Создаст `run.sh` — обёртку для запуска.
5. Добавит cron-задачу, которая по будням с 09:00 до 18:00 каждые 30 минут
   снимает текущие лимиты через `/usage`.
6. Создаст ярлык «Claude Usage» в меню приложений и добавит его в автозапуск.

> Для установки системных пакетов скрипт использует `sudo` — он спросит пароль.

## Запуск

- Через меню приложений → **Claude Usage**, или
- запустив `~/.local/share/claude-widget/run.sh`, или
- автоматически при следующем входе в систему (autostart).

Для запуска из исходников без установки: `python3 app.py` (нужен установленный
`pywebview` и GTK/WebKit).

## Как собираются данные

Есть два независимых источника:

- **Токены** читаются прямо из локальных логов сессий Claude Code
  (`~/.claude/projects/**/*.jsonl`). Это бесплатно и происходит мгновенно.
  Завершённые дни сохраняются в `~/.claude-widget/token_history.json`, поэтому
  история не теряется, даже когда Claude Code удаляет старые логи.
- **Лимиты** (кольца) берутся из вывода команды `/usage`. Чтобы их получить,
  виджет запускает настоящий процесс `claude` в фоне. **Это расходует квоту вашей
  сессии**, поэтому обновление лимитов делается не постоянно, а:
  - по расписанию cron (будни 09:00–18:00, каждые 30 минут);
  - вручную — кнопкой **«Обновить»** в шапке виджета.

## Полезно знать

- «День» в статистике считается с **09:00 до 09:00** локального времени, а не
  календарные сутки.
- Окно держится поверх других; тема (светлая/тёмная) запоминается между запусками.
- «Входные токены» включают кеш — это эффективный размер промпта, а не только
  «сырой» input.

## Диагностика

- **Окно не открывается / белый экран** — не установлены биндинги WebKit2GTK.
  Проверьте, что есть пакет `gir1.2-webkit2-4.1` (или `-4.0`), и переустановите.
- **Кольца показывают «—» или N/A** — виджет не смог снять `/usage`. Убедитесь,
  что команда `claude` запускается и вы залогинены. Сырой вывод последней попытки
  лежит в `~/.claude-widget/usage_debug.txt`.
- **`claude: command not found` в логах cron** — CLI не в `PATH`. Положите его в
  `~/.local/bin/claude` или поправьте окружение в cron-задаче.
- **Логи запуска по cron** — пишутся в `~/.claude-widget/usage_cron.log`.

## Удаление

```bash
# убрать cron-задачу
crontab -l | grep -v 'claude-widget-usage' | crontab -

# удалить ярлыки и приложение
rm -f ~/.local/share/applications/claude-widget.desktop
rm -f ~/.config/autostart/claude-widget.desktop
rm -rf ~/.local/share/claude-widget
```

Все данные виджета лежат в одном каталоге — `~/.claude-widget/` (история токенов,
снимки лимитов, debug и cron-лог). Удалите его, если хотите стереть накопленное.
