# CLAUDE.md

Руководство для Claude Code при работе с этим репозиторием.

## Обзор

`claude-widget` — десктопный виджет (pywebview, поверх других окон) для Linux,
который показывает лимиты использования Claude Code (кольца) и статистику токенов
(плитки, разбивка по моделям, графики по дням). Данные берутся локально из логов
сессий Claude Code и из вывода команды `/usage`.

## Архитектура

- **`app.py`** — точка входа. Создаёт окно pywebview 452×720 и пробрасывает
  Python-методы в JS через класс `Api`: `get_rings`, `get_tokens`,
  `get_history_data`, `refresh_usage`. При старте в фоновом потоке вызывает
  `_seal_history()`, чтобы завершённые дни попали в постоянное хранилище даже
  если вкладку «История» не открывали.
- **`tokens.py`** — парсит `~/.claude/projects/**/*.jsonl`, агрегирует токены.
- **`usage.py`** — скрапит проценты лимитов из `/usage` через PTY.
- **`web/`** — фронтенд на чистом JS (`app.js`, `index.html`, `style.css`).
- **`install.sh`** — установщик (apt-зависимости, venv, cron, .desktop, autostart).

JS вызывает Python через `window.pywebview.api.*` (см. `web/app.js`, функция
`api()` и `boot()`).

## Ключевые концепции

- **Логический день**: сутки считаются от 09:00 до 09:00 локального времени
  (`RESET_HOUR = 9` в `tokens.py`), а не календарные. Вся группировка токенов и
  меток дат идёт по этой границе.
- **Эффективный вход**: реальный промпт лежит в кеш-полях, поэтому «входные
  токены» = `input + cache_read + cache_creation` (`_eff_in` в `tokens.py`).
- **Запечатывание истории**: Claude Code ротирует старые `.jsonl`, поэтому
  завершённые дни сохраняются в `~/.claude-widget/token_history.json`. Дни, всё
  ещё присутствующие в логах, считаются точнее и перезаписывают хранилище.
- **Кеш файлов**: `_FILE_CACHE` в `tokens.py` хранит распарсенные записи по
  `(mtime, size)`, чтобы не перечитывать неизменённые логи.
- **Скрейп `/usage` тратит квоту**: `usage.py` запускает настоящий процесс
  `claude` в псевдотерминале, проходит экраны trust/tips, отправляет `/usage`,
  чистит ANSI и достаёт проценты регэкспами. Каждый вызов расходует квоту сессии.

## Расположение данных

- `~/.claude/projects/**/*.jsonl` — логи сессий Claude Code (источник токенов).
- `~/.claude-widget/token_history.json` — запечатанная история токенов по дням.
- `~/claude_usage_log.json` — снимки процентов лимитов (пишет `usage.py`).
- `~/claude_usage_debug.txt` — сырой вывод последнего скрейпа `/usage` (для отладки).

## Команды

```bash
python3 app.py        # запустить виджет (нужен pywebview + GTK/WebKit)
python3 usage.py      # разовый скрейп /usage (в будни 9–18; пишет в лог)
python3 tokens.py     # самотест: печатает агрегаты токенов/историю/серии
./install.sh          # полная установка (зависимости, venv, cron, ярлык)
```

## Конвенции

- Фронтенд строит DOM без `innerHTML` — только через хелперы `h()` / `svgEl()`
  и `textContent` (защита от инъекций). Сохраняйте этот подход.
- Все строки UI — на русском.
- Зависимости Python устанавливаются в `install.sh` (pywebview в venv с
  `--system-site-packages`); отдельного `requirements.txt` нет.
